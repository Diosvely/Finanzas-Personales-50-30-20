## 👤 Contexto: quién hay detrás de este repositorio

Soy **Diosvely Pérez Arteaga**, Controller Financiero con más de 15 años de experiencia en análisis de costes, control de gestión e informes financieros.

Estoy en proceso de transición profesional hacia roles de **Data Analytics y Economía en España**, y este repositorio es parte de esa demostración pública.

Mi perfil es híbrido: entiendo los datos porque he vivido en ellos desde el lado financiero. Ahora estoy construyendo las herramientas técnicas para analizarlos de forma más escalable.

> **Certificación:** Microsoft Power BI PL-300  
> **Stack actual:** Excel · Power Query · Power Pivot · DAX · SQL · Power BI

---

## 🎯 Por qué existe este proyecto

La mayoría de los proyectos de portafolio muestran *qué* sabes hacer.  
Este quiere mostrar *cómo piensas*.

Elegí las finanzas personales como caso de uso por tres razones:

1. **Es un dominio que conozco en profundidad** desde el lado profesional — no estoy simulando, estoy aplicando lógica real de FP&A a escala doméstica.
2. **Es universal** — cualquier persona puede entender el problema, lo que hace la solución más fácil de evaluar.
3. **Escala bien** — el mismo modelo que empieza en Excel puede evolucionar hacia un sistema analítico completo con SQL y Power BI.

## 🗺 Estructura de la serie — 3 módulos, 1 evolución

La serie documenta una evolución técnica real, desde la hoja de cálculo hasta el modelo analítico:

```
📁 01-presupuesto/      ← Estás aquí
📁 02-datos-reales/     ← Próximamente

```

### 📁 01 — Presupuesto en Excel
**Pregunta:** ¿Cómo estructuro datos financieros con lógica de modelo, no solo con fórmulas?

- Diseño de estructura de datos normalizada
- Fórmulas avanzadas: `SI.ERROR`, `BUSCARX`, acumulados YTD
- KPIs financieros con formato condicional (semáforo)
- Automatización con VBA
- Dashboard de visualización

**Lo que demuestra:** pensamiento estructurado, separación de datos y lógica, diseño escalable.

---

### 📁 02 — Datos Reales con Power Query + Modelo en Estrella
**Pregunta:** ¿Cómo convierto datos bancarios reales en un modelo analítico?

- Importación y limpieza con Power Query
- Clasificador automático de transacciones
- Modelo en estrella con Power Pivot
- Paso de OLTP a OLAP

**Lo que demuestra:** capacidad de transformar datos sucios en información estructurada.

---

### 📁 03 — Análisis con DAX y Desviaciones
**Pregunta:** ¿Cuánto me desvié del plan y por qué?

- Medidas DAX para comparación Plan vs Real
- Tablas dinámicas avanzadas
- Dashboard financiero profesional
- Análisis de varianzas

**Lo que demuestra:** pensamiento analítico, narración con datos, toma de decisiones.

## 🧠 La metodología: regla 50/30/20

No elegí esta regla porque sea la más conocida.  
La elegí porque es **verificable, comparable y fácil de convertir en KPI**.

| Categoría | Límite | En este modelo |
|-----------|--------|----------------|
| Gastos Fijos | ≤ 50% | Vivienda, suministros, seguros, financiación |
| Gastos Variables | ≤ 30% | Alimentación, ocio, transporte, compras |
| Ahorro / Utilidad | ≥ 20% | Lo que queda después de vivir bien |

Cada módulo del proyecto valida si el comportamiento financiero real cumple estos umbrales.
